Projects

StudyHub 2

A web platform I designed and built for York School which allows students to share, organize, and access educational study resources school-wide. It's used by over 80% of York students and faculty to share and learn. Version 2 is built on Angular 6 and uses Firebase, Algolia Search, and Google Cloud Functions as a backend. StudyHub has evolved continuously through many prototypes where I switched to new server stacks, made the web app fully responsive, mobile-ready and offline capable, and then completely rebuilt Studyhub after I asked Google employees to review the security of my website. It took me nearly two years to complete the first version and I spend another six months totally reworking the codebase to be much faster and more feature-rich for version 2. You can learn more about StudyHub's development on my dev blog and check out the live version at studyhub.york.org (if you would like a temporary login, email me).

Visualization of Mock Seafloor Mapping Data

I built this activity for the Seafloor Science and ROV Summer Camp that I work at over the summer. It's designed to teach kids how seafloor maps are made in the open ocean. It's a two-part project, the first being a robot fitted with a laser rangefinder, that "scans" a clay terrain model by driving on a glass sheet above the model and measuring distances. In the second part, camp participants paste the data they've collected into my website to see the model became a virtual seafloor map. This activity mimics how autonomous underwater vehicles and side-scanning boats create seafloor maps using sonar. It's built on WebGL using THREE.js and uses a custom GPU shader to render the depth-mapped color codes. You can play with it at kw-m.github.io/ssrov-heightmapper and look at the source code on my Github.

Mobile VR Games with Google Cardboard

I started this project to try and create a real VR mobile game using the Unity game engine and the Google Cardboard SDK. I was inspired by games like Ball Resurrection to do an aerial ball platformer game but in VR. Instead of regular level design, I thought it would be cool if the game procedurally generated each level, so I made a C# script to generate levels in-game. In my first prototypes, I modeled a stone pathway that would crumble and fall into the sea behind the player; however, the physics behind the crumbling pathway proved too costly for a mobile CPU, so I switched to a simpler pathway design. I finally showcased my work at York's art fair, where people could try out my game and a watch a 360 video a friend made in VR. If you have an Android device, you can play Prototype 2, by installing the app's APK or application package.
